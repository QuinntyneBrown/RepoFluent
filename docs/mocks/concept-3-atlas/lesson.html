Glossary

pricing rule

A pricing rule is a discrete promotion or adjustment that can inspect cart contents, customer context, and subtotal to produce a discounted total. In Orion, rules are registered through PromotionRuleRegistry.

CartPricingService.cs

src/Orion.Checkout/Services/CartPricingService.cs

Snapshot: main @ 4f2c9ab · captured Jul 14, 2026 — repository may have changed since capture. 

public sealed class CartPricingService : ICartPricingService
{
    private readonly IPromotionRuleRegistry _rules;
    private readonly ITaxCalculator _tax;

    public CartPricingService(IPromotionRuleRegistry rules, ITaxCalculator tax)
    {
        _rules = rules;
        _tax = tax;
    }

    public async Task<CartTotals> PriceAsync(Cart cart, CancellationToken ct)
    {
        var subtotal = cart.Lines.Sum(l => l.UnitPrice * l.Quantity);
        var discounted = await _rules.ApplyAsync(cart, subtotal, ct);
        var tax = _tax.Calculate(discounted, cart.ShippingAddress);
        return new CartTotals(subtotal, discounted, tax);
    }
}

Step 2 walkthrough

PriceAsync first aggregates cart lines into a subtotal, then delegates promotion application to _rules.ApplyAsync. Tax is calculated after discounting so the returned CartTotals keeps the original subtotal, discounted amount, and tax separate for the Angular UI.

Inline knowledge check

Which component is responsible for registering promotion rules?

Correct answer: PromotionRuleRegistry. It composes available promotion rules before CartPricingService applies them.
