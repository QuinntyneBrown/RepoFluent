# Assessment and Mastery — L1 Requirements

## Document control

| Field | Value |
| --- | --- |
| Subsystem | Assessment and Mastery (ASM) |
| Status | Draft production requirement |
| Source | PRD 4, 7.2, 8.7, 10, 11, and pilot acceptance criteria 4/5/7 |
| Owners | Learning Science and Assessment Platform |

## Mission and boundary

The subsystem administers formative quizzes and summative tests, protects answer material, selects items, enforces attempt policy, grades deterministic responses, records immutable evidence, and derives explainable objective-level mastery. It does not provide external certification, employee performance ranking, or AI grading without a deterministic contract-approved method.

## High-level requirements

| ID | Priority | Requirement | PRD trace |
| --- | --- | --- | --- |
| L1-ASM-01 | Must | Assessments shall support single-choice, multiple-choice, true/false, ordering, matching, and deterministically gradable short-answer items. | ASM-01 |
| L1-ASM-02 | Must | Formative quizzes and summative tests shall enforce their configured feedback-release behavior. | ASM-02 |
| L1-ASM-03 | Must | Assessments shall enforce question pools, selection/randomization, passing thresholds, attempt limits, and time limits as configured. | ASM-03 |
| L1-ASM-04 | Must | Every scored item shall map to one or more declared learning objectives, systems, or subsystems. | ASM-04 |
| L1-ASM-05 | Must | Every submitted attempt shall create immutable, auditable evidence with curriculum/item versions, responses, score, timestamps, and outcome. | ASM-05 |
| L1-ASM-06 | Must | Answer keys and protected explanations shall remain inaccessible until their configured release point. | ASM-06 |
| L1-ASM-07 | Must | Learners shall receive scoring, timing, attempt, completion, and feedback rules before an attempt begins. | ASM-07 |
| L1-ASM-08 | Should | Authorized reviewers should invalidate defective questions and recalculate affected outcomes transparently. | ASM-08 |
| L1-ASM-09 | Should | Content-quality checks should detect duplicates, answer leakage, missing rationales, and objectives without assessment coverage. | ASM-09 |

## Quality requirements

- Deterministic grading shall produce the same result for the same versioned item, response, and grading-policy version.
- The server shall be the authority for eligibility, timing, item selection, submission, and release state.
- Learner-facing assessment UI shall meet WCAG 2.2 AA and remain usable on narrow supported viewports and without GPU features.
- Telemetry shall not log protected answers or free-text response contents by default.
- Published item versions and original attempt evidence shall never be rewritten by recalculation.

## Dependencies and exclusions

- Depends on Curriculum Contract/Lifecycle, Identity/Access, Learning Experience, Analytics, Audit, and reliable time/persistence services.
- Proctoring, high-stakes external certification, nondeterministic free-form grading, and hidden employee ranking are outside initial scope.
