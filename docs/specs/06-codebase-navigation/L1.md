| Field | Value |
| --- | --- |

| Status | Draft production requirement |

| Owners | Code Learning and Curriculum Platform |

## Mission and boundary

The subsystem connects learning explanations to authorized, revision-aware source context. It owns read-only excerpt rendering, deep-link resolution, code tours, source snapshot/drift presentation, access filtering, optional file/symbol navigation, and supplied architecture relationship visualization. It is a learning navigator, not an IDE, source editor, static-analysis engine, or autonomous repository client.

## Quality requirements

- Source context shall retain curriculum, version, lesson, reference, and snapshot identifiers.
- Large files/trees shall use progressive loading or virtualization while meeting interaction budgets.
- Excerpts and links shall be sanitized, classification-aware, and absent from unauthorized telemetry.
- All code and relationship functions shall have keyboard-accessible, non-canvas representations.

## Dependencies and exclusions

- Depends on contract source/reference metadata, repository classification policy, learning context, syntax libraries, and system-map presentation.
- Current-provider integration, full repository browsing, static analysis, editing, running, debugging, and source modification are outside MVP unless separately approved.
