# Agent Authoring Kit — L1 Requirements

## Document control

| Field | Value |
| --- | --- |
| Subsystem | Agent Authoring Kit (AAK) |
| Status | Draft production requirement |
| Source | PRD 3, 4, 7.1, 8.4, 11, and pilot acceptance criterion 1 |
| Owners | Developer Experience and Curriculum Platform |

## Mission and boundary

The subsystem enables organizations to generate contract-conformant curriculum packages inside an approved environment using an agent of their choice. It owns the distributable authoring bundle, agent instructions, reusable skills/prompts, ecosystem guidance, local validation command, examples, and generation manifest. It does not own agent runtime access, customer repository permissions, or publication approval.

## High-level requirements

| ID | Priority | Requirement | PRD trace |

## Quality requirements

- A kit version shall declare compatible contract and validator versions and shall be reproducible from its release manifest.
- Instructions shall use testable language and shall separate mandatory package rules from optional authoring guidance.
- The local validator shall provide deterministic exit status and machine-readable as well as human-readable output.
- The kit shall be usable from documented supported environments without hidden network dependencies.
- Examples shall contain no real customer source, secrets, personal data, or protected answer material.

## Dependencies and exclusions

- Depends on the Curriculum Input Contract release unit and security-approved agent operating guidance.
- Agent/model selection, commercial terms, and customer-side repository access are outside RepoFluent MVP.
- Managed generation is deferred unless separately approved.
