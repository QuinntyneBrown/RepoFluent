## Detailed requirements

### L2-CBN-01 — Safe excerpt rendering

**Traces to:** L1-CBN-01, L1-CBN-09; CODE-01

The renderer shall display supplied C# and Angular-family excerpts as inert text with language-appropriate syntax highlighting, preserved line numbers/ranges, wrapping or horizontal scrolling, copy policy, and accessible text semantics. It shall never interpret excerpt markup or execute source.

**Acceptance criteria**

- **GIVE** an authorized excerpt containing HTML/script-like text, Unicode, long lines, and highlighted ranges
- **WHEN** the learner opens it
- **THEN** exact allowed text is displayed inertly with correct lines/highlights, no script executes, and the excerpt is readable by keyboard and assistive technology

### L2-CBN-02 — Reference identity and resolution

**Traces to:** L1-CBN-02; CODE-02

A code reference shall resolve by tenant, curriculum/version, declared repository, repository-relative path, optional revision, symbol, and line/range. Resolution shall validate all anchors, normalize paths without traversal, and return a safe unavailable state for missing or invalid targets.

**Acceptance criteria**

- **GIVE** a valid reference to `src/Order.Api/Controllers/OrderController.cs`, symbol `Create`, lines 42–57
- **WHEN** an authorized learner follows the lesson link
- **THEN** the matching reference opens with its exact version/snapshot and originating lesson context

- **GIVE** a path containing traversal outside the declared repository
- **WHEN** reference resolution runs
- **THEN** access is rejected, no filesystem or repository content is disclosed, and a security-relevant event is recorded

### L2-CBN-03 — External deep-link policy

**Traces to:** L1-CBN-02, L1-CBN-05

If a repository provider link is supplied, the subsystem shall construct or allow only approved provider schemes/hosts and authorized repository/path/revision data. The learner shall be warned before leaving RepoFluent as configured. Tokens or credentials shall never appear in URLs.

**Acceptance criteria**

- **GIVE** an approved provider mapping and an authorized immutable reference
- **WHEN** the learner opens the provider link
- **THEN** the URL targets the declared repository/revision/path without credentials and follows the configured external-navigation notice

### L2-CBN-04 — Code tour model and resume

**Traces to:** L1-CBN-03; CODE-03

A code tour shall maintain tour identity/version, ordered step, originating course/lesson, reference selection, learner guidance, and completed-step state. Moving among steps and returning from related views shall preserve this context across sessions where progress policy permits.

**Acceptance criteria**

- **GIVE** a four-step Angular-to-C# code tour and a learner on step 3
- **WHEN** the learner opens the system map, signs out, then resumes on another device
- **THEN** the tour restores step 3 with the correct lesson/version and lets the learner navigate previous/next steps in published order

### L2-CBN-05 — Source snapshot presentation

**Traces to:** L1-CBN-04; CODE-04

Every excerpt/reference shall present available repository identity, branch/commit or document revision, and snapshot time. The UI shall distinguish immutable binding, branch-only binding, and unknown/current state and shall not label an unverified reference as current.

**Acceptance criteria**

- **GIVE** one reference pinned to a commit and another containing only a branch snapshot
- **WHEN** they are displayed
- **THEN** the pinned reference shows its immutable revision and the branch-only reference shows a clear potential-drift state

### L2-CBN-06 — Drift warning

**Traces to:** L1-CBN-04; CODE-04

When current authorized provider metadata is available, the subsystem shall compare the published snapshot/revision to current state using documented semantics. A mismatch or inability to confirm shall produce a nonblocking warning without rewriting the published reference.

**Acceptance criteria**

- **GIVE** a published reference bound to commit A and an authorized provider reporting current commit B
- **WHEN** the learner views the reference
- **THEN** the original A content/version remains identified and a visible accessible drift warning states that current repository state may differ

### L2-CBN-07 — Access re-evaluation

**Traces to:** L1-CBN-05; CODE-05

Authorization shall be re-evaluated when resolving an excerpt, tour step, file/symbol node, provider link, search result, or relationship. Previously saved progress or bookmarks shall not confer content access. Unauthorized references shall not expose path, symbol, excerpt, relationship, or existence beyond safe messaging.

**Acceptance criteria**

- **GIVE** a learner bookmarked a reference before its repository classification was raised
- **WHEN** the learner opens the bookmark after losing access
- **THEN** the reference is denied without excerpt/path disclosure and the learning view offers a safe return path

### L2-CBN-08 — File tree navigation

**Traces to:** L1-CBN-06; CODE-06

Where imported metadata is present, a file tree should expose permitted repository-relative hierarchy with expandable nodes, file type, and links to known references. It shall progressively load or virtualize large structures and shall exclude undeclared/unclassified paths.

**Acceptance criteria**

- **GIVE** imported metadata for a large repository with one restricted subtree
- **WHEN** an authorized learner navigates and filters the tree
- **THEN** permitted nodes remain responsive, known references open in context, and the restricted subtree is absent

### L2-CBN-09 — Symbol relationships

**Traces to:** L1-CBN-06; CODE-06

Supplied symbol metadata should support declared symbols, containing file/range, kind, and typed relationships. The interface shall label relationships as supplied snapshot metadata and shall not claim complete static-analysis coverage.

**Acceptance criteria**

- **GIVE** package metadata declares a controller method and its relationship to a command handler
- **WHEN** the learner inspects the symbol
- **THEN** the declared relationship is shown with source/version provenance and is not labeled as a live or exhaustive call graph

### L2-CBN-10 — Architecture relationship views

**Traces to:** L1-CBN-07; CODE-07

The subsystem should visualize supplied calls, dependencies, routes, components, and data flows with typed/directed relationships, selected-node details, provenance, and links to related learning/code. Every visualization shall provide an accessible list or table with equivalent information and controls.

**Acceptance criteria**

- **GIVE** a supplied data flow from Angular Storefront to Order API to message bus and processor
- **WHEN** a keyboard-only learner selects each relationship using the accessible representation
- **THEN** the same nodes, directions, types, provenance, and related links available visually are available without canvas or pointer input

### L2-CBN-11 — Current provider connection

**Traces to:** L1-CBN-08; CODE-08

A future provider connection may retrieve current metadata or read-only source only after tenant configuration, user/service authorization, repository allow-listing, classification checks, and audit. Provider failure shall not make the published supplied excerpt unavailable.

**Acceptance criteria**

- **GIVE** a provider integration is configured but temporarily unavailable and a published package contains an approved excerpt
- **WHEN** the learner opens the reference
- **THEN** the published excerpt remains usable with snapshot context, current-state lookup reports a safe warning, and no write operation is attempted

### L2-CBN-12 — Non-execution and non-modification boundary

**Traces to:** L1-CBN-09

The subsystem shall expose no source edit, commit, execution, build, deployment, or write-provider operation. Relationship statements not present in approved package/provider metadata shall be labeled as unavailable rather than inferred.

**Acceptance criteria**

- **GIVE** source content or metadata contains commands, scripts, or executable examples
- **WHEN** it is rendered or navigated
- **THEN** it remains inert read-only content and no API/UI control can execute or persist changes to the customer repository
