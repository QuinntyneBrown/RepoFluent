## Mission and boundary

The subsystem defines the authoritative, portable, versioned exchange boundary between content-producing agents and RepoFluent. It owns the JSON Schema, Interface Control Document (ICD), canonical formats, compatibility rules, validation vocabulary, safe extension mechanism, and contract fixtures. It does not execute agents, publish curricula, or render learner content.

## Quality requirements

- Schema and ICD semantics shall not contradict; automated release checks shall detect drift where machine-verifiable.
- Validation of the same bytes against the same contract version and validator release shall produce the same normalized issue set.
- Contract artifacts shall be distributable for validation inside a customer's approved environment.
- Error output shall not echo secret-like values or unnecessary source excerpts.
- Contract evolution shall favor additive compatible changes within a major version.

## Dependencies and exclusions

- Depends on product ownership for contract versioning and security review of supported content types.
- Supplies artifacts to the Agent Authoring Kit and validation behavior to Curriculum Lifecycle.
- Direct repository ingestion and model-specific agent formats are outside the contract boundary for MVP.
- Package scale limits, additional locales, media formats, and supported-version window require confirmation before production baseline.
