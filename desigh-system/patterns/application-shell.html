Routing and context

  • Every primary document has a stable, shareable URL scoped to tenant, curriculum version, and permitted object.
  • Tabs reflect open work but do not replace browser history. Back/forward restores the prior route and selection.
  • Deep links restore the closest available workspace state and explain missing permission, retired version, or stale source.
  • Split-pane widths and sidebar collapse may persist per user; critical content order remains encoded in the DOM.

Device classes

WIDE / ≥1024
Persistent titlebar, activity bar, sidebar, tabs, statusbar, and available split panes.
COMPACT / 768–1023
Collapsible sidebar. Inspectors may stack below content while open tabs remain scrollable.
PHONE / <768
Drawer navigation, active-view title, stacked code/inspector, fixed destination bar, safe bottom spacing.

Phone adaptations preserve assessments, lesson completion, source excerpts, map relationships, and validation recovery. Decorative statusbar details may collapse.

Shell-level failure modes

CONDITION BEHAVIOR
WebGPU unavailable Load accessible HTML/SVG system map without blocking navigation
Offline after load Keep permitted cached content, mark sync unavailable, queue retry-safe progress
Session expired Preserve local context, authenticate, then return to the same route
Unauthorized deep link Show a safe denial without confirming hidden content metadata
Global load failure Show error identifier, retry, support route, and non-sensitive diagnostics
