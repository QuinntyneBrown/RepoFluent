Component / C12

Data visualization

Charts reveal distributions, comparisons, trends, and gaps while keeping population, time window, filters, versions, and exact values available.

RF-C12

Bar chart anatomy

OBJECTIVE MASTERY / CHECKOUT TEAM / CURRICULUM V3
  1. Title states the measure, not merely the chart form.
  2. Subtitle states population, aggregation, and time window.
  3. Plot uses fixed palette slots and visible exact values.
  4. Legend and accessible table use the same series order.

Choose by analytical task

TASK FORM REPOFLUENT EXAMPLE
Compare categories Bar / dot plot Mastery by objective
Show distribution Histogram / box plot Assessment result distribution
Show trend Line chart Weekly completion over time
Part-to-whole Stacked bar Assignment states within one cohort
Exact comparison Table Learner attempt history
Relationships Node-link map + table System and subsystem topology

Do not use a donut for more than three parts, a gauge for a distribution, or a spatial map when a table communicates the task more directly.

Privacy and interpretation

  • Label hidden or suppressed groups without revealing a sub-threshold population.
  • Never imply causation from correlation or compare cohorts without labeling population and version differences.
  • Explain calculation inputs and whether the measure uses latest, best, average, or valid attempts.
  • Use “No data” separately from zero; show stale, partial, or delayed data explicitly.

Usage rules

DO

Start with the question, show exact values, disclose filters, and provide a table.

DON'T

Create decorative dashboards, hidden rankings, unlabeled axes, or color-only series.
