Command palette & menus

The palette is a fast global route to permitted content and actions. Menus hold local, secondary commands without becoming the only way to discover essential work.

Placement and permissions

  • Anchor local menus to their trigger and keep them within the viewport.
  • Open from keyboard directly below the trigger; preserve the same item order as pointer use.
  • Do not render results the user cannot know exist. A visible disabled item is appropriate only when its existence teaches a permission or workflow rule.
  • Never expose protected assessment answers, unauthorized source paths, or cross-tenant content through search indexes or result counts.
DO

Keep frequent commands visible in context and offer the palette as an accelerator.

DON'T

Hide required navigation exclusively inside a “more” menu or expose unauthorized result metadata.
