Validation and sensitive input

  • Validate locally when deterministic; preserve entered values when the server rejects a submission.
  • Place a form summary before the first field after a failed submit and link each error to its control.
  • Never expose answer keys, tokens, secrets, or protected explanations through field defaults or client-side source.
  • Confirm destructive settings with a dialog that names the affected tenant, curriculum, or record set.
