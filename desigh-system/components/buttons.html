Component / C01

Buttons

Buttons start immediate actions. Their hierarchy communicates consequence and priority before the learner reads the label.

RF-C01

Variants

DEFAULT STATES
VARIANTUSELIMIT
PrimaryPreferred next action in the current regionOne per action group
SecondaryValid alternative or neutral operationMay appear in groups
GhostDismissal and low-emphasis local actionNot for destructive action
DangerDestructive or irreversible intentRequires consequence language

States

REST / PRESSED / DISABLED / BUSY

Disabled controls remain discoverable only when the surrounding interface explains why. Prefer an enabled control that validates and explains over a silent disabled action.

Sizes and icon buttons

28 / 36 / 44 / ICON

Icon-only controls require an accessible name and a tooltip on hover and focus. Small buttons belong inside toolbars with a 44px group target on touch devices.

Toggle groups

EXCLUSIVE VIEW MODE

Use a toggle group when a view changes in place and every option remains available. Use tabs when each option owns a distinct content panel and keyboard arrow behavior.

Anatomy and placement

  1. Optional leading icon names the action category.
  2. Verb-first label describes the immediate result.
  3. Optional shortcut reinforces keyboard availability.
  4. Border and fill establish hierarchy and state.
DO

Use “Publish version,” “Export CSV,” and “Mark complete.”

DON'T

Use vague labels such as “OK,” “Yes,” “Submit,” or more than one primary action.
