Toolbars & tabs

Toolbars operate on the visible object or view. Tabs preserve a small working set of related documents without replacing route-level navigation.

Tab states

STATE VISUAL BEHAVIOR
Active Canvas surface, strong ink, green top edge Associated panel is visible
Open Surface 1, muted ink Retains document state but panel is hidden
Modified Small dot after label Closing requests confirmation when loss is possible
Stale Amber icon plus written tooltip Opens revision comparison or warning
Overflow Horizontal scroll + tab list menu Every open tab stays keyboard reachable

Responsive behavior

  • Keep the primary contextual action visible before moving actions into overflow.
  • Let tabs scroll horizontally on compact tablet widths; never shrink labels into unreadable fragments.
  • On phone widths, replace document tabs with the active title and an “Open views” list.
  • Mode switches stay adjacent to the content they alter and persist independently per document.
