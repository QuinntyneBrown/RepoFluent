Sorting, filtering, and selection

  • Sortable headers use a button with the current direction announced through aria-sort.
  • Filters appear above the table, show active criteria as removable chips, and announce the new result count.
  • Row selection is separate from row navigation. Use checkboxes for multi-select and an explicit primary link for drill-down.
  • Keep bulk actions disabled until selection exists, then announce how many records they affect.

Pagination and responsive behavior

  • Paginate server-backed records; virtualize only when selection and assistive-technology behavior remain predictable.
  • Keep identity and primary state columns sticky on wide tables; allow horizontal scrolling inside the table region.
  • On phones, transform only simple two- or three-column tables into labeled records. Preserve true comparison tables as scrollable tables.
  • Exports reflect active filters and authorized columns, and label the population, time window, and curriculum version.
