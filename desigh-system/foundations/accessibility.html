Foundation / F07

Accessibility

WCAG 2.2 AA is the minimum product contract. Keyboard access, semantic structure, text alternatives, and equivalent representations are designed in—not audited in later.

Focus order and restoration

VISIBLE FOCUS / LOGICAL ORDER
1 · Read semantics
  • DOM order matches visual and reading order at every breakpoint.
  • Opening a dialog moves focus inside it; closing returns focus to the exact trigger.
  • Route changes place focus on the page heading or main region and announce the new context.
  • Focus is never removed, made transparent, or represented by color change alone.

Semantic structure

NATIVE FIRST
  • button for actions; a for navigation
  • table for relational data
  • fieldset and legend for choices
  • details for simple disclosure
ANNOUNCEMENTS
  • Polite live regions for save and filter results
  • Assertive alerts only for blocking failure
  • Progress names include the work being measured
  • Errors link to the affected field

Equivalent representations

SYSTEM MAP
Keyboard-operable nodes, inspector details, and a complete relationship table.
CHARTS
Visible labels, a summary, and an accessible data table with the same filters.
CODE
Text remains selectable; line emphasis is announced and does not disrupt reading.

Content and state

DO

Name the state and next action: “Import failed. Fix 3 schema errors, then validate again.”

DON'T

Say “Invalid,” rely on a red border, or hide recovery instructions in a tooltip.

Release checklist

  • Complete the flow with keyboard only at 100% and 200% zoom.
  • Verify landmarks, headings, names, roles, values, descriptions, and live updates with a screen reader.
  • Test default, hover, focus, active, disabled, loading, empty, error, success, stale, and restricted states.
  • Test high contrast, reduced motion, narrow viewport, long translated labels, and text-only alternatives.
  • Record and fix automated issues, then complete manual task-based testing.
