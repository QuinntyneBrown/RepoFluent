Foundation / F06

Motion

Motion confirms a relationship or state change. It stays short, interruptible, and unnecessary for comprehension.

RF-F06

Motion tokens

Token Duration Use
motion-fast 100ms Hover, press, color, and border feedback
motion-base 180ms Tabs, drawer, disclosure, and small state changes
motion-slow 280ms Dialog entry and spatial transitions only
ease-standard .2, 0, 0, 1 Elements entering or changing state
ease-exit .4, 0, 1, 1 Elements leaving the interface

Allowed motion

ORIENTATION
A drawer slides from the edge it belongs to. A selected tab changes in place.
FEEDBACK
A button depresses one pixel. A saved toast enters without blocking work.
RELATIONSHIP
A code reference highlights the exact lines opened from a lesson.

Ambient effects may enrich a system visualization, but must stop when the visualization is not active and must never indicate required state.

Loading and progress

SKELETON / CONTENT SHAPE
  • Use a skeleton only when the final structure is known and likely to arrive quickly.
  • Use determinate progress for imports, uploads, and assessment submission when measurable.
  • Use written status plus an activity indicator for unknown waits; announce significant changes through a live region.

Reduced motion

System preference is authoritative

Under prefers-reduced-motion: reduce, durations collapse to 1ms, smooth scrolling stops, shimmer runs once, and spatial movement becomes an immediate state change.

The reduced version must preserve focus movement, announcements, final state, and control availability.

Usage rules

DO

Animate opacity, transform, and color sparingly, then test interruption halfway through.

DON'T

Auto-pan a map, loop decorative motion indefinitely, or delay navigation until an animation completes.
